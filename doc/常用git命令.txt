小组git仓库使用规范
1.  code文件夹：存放项目前端，后端代码
    doc文件夹：学习日志，项目文档，另外，最新的项目文档在项目文档列表上传
2.  每个人把master分支clone到本地之后，新建一个自己的分支，之后只需提交自己的分支，完成某部分功能后，在线上发起合并分支请求，评审人与合并人选择除助教外的所有人


常用git命令

新建或修改文件


git add .                   #将文件加入暂存区,'.'表示所有文件
git add filename            #将filename加入暂存区

git commit -m <提交信息>      #将暂存区文件加入本地仓库

git push                    #将本地仓库推送至远程仓库

git status                  #查看暂存区文件

git pull                    #更新本地仓库

git branch <分支名称>       #本地新建分支

git checkout <分支名称>     #切换分支


git checkout -b <分支名称>  #新建并切换到新分支

git branch                  #查看本地分支

git branch -r               #查看远程分支

git branch -a               #同时查看本地远程分支

touch <filename>            #新建文件

git push --set-upstream origin <featurename>
                            #提交本地分支

删除文件
git rm filename
git commit -m <提交信息>
git push
